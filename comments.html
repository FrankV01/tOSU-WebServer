Welcome

Welcome to my Web Server project. I hope you have a great visit. This document contains some quick notes on building and running the programs.

Compiling

A build can be accomplished one of two ways. The first is manually running javac the second is to use ant. The two methods are outlined below.

javac

The is a single file that must be compiled Any dependencies should be compiled inherently.

The file is:

  • WebServer.java

To compile the file and all related files, simply issue the following command:

  • javac WebServer.java

Ant [Optional]

If you desire, you can try to use ant to build the files. Using ant saves some typing but it truly isn't required.

I've provided a build file called build.xml. Simply point your command window to the directory you unzipped the files at. And then execute ant. A basic build will be done and the resulting class files will be stored in "build".

Further ant build options can be explored via the ant -projecthelp command. This will simply list all build targets but the default is the most commonly used one.

Running the Web Server

Execution of the program is very simple and straight forward. There are some command line options and those are laid out below along with the default values. The web server, by default, runs on port 2540 as required by the assignment.

For simplicity's sake, here are some example executions along with an explanation

Start up Command Description
java WebServer

Most simplistic execution. Starts a web server serving files from the directory that the class file resides in. No debugging information is outputted.
java WebServer -l

Start the sever in Listener mode. This uses MyListener.java
java WebServer -d -f=../pages

Start the web server with debug mode enabled. For files, look one directory back in another directory called pages.

The complete command line option descriptions

Command Switch Command Name Default Description
-? Help N/A Lists the programs optional switches and a bit of information on how to use the program.
-p Server Port 2540 The port to serve from.
-d Enable Debug Mode Off Enable debug mode which simply prints out more detailed status messages.
-l MyListener Off Execute the server utilizing the MyListener.java class file. This turns the server in to the Client Lister which was used to gather the client output.
-f Serve-From-Directory . (e.g. the current directory) This is the directory from which the HTML file to serve come from. The default is the directory the server is running from. You can use either relative or absolute paths with this option.

As an option, to serve files from C:\wwwroot\pages\ you'd use the option as: -f=C:\wwwroot\pages.
Start up checks are performed to loosely validate the input.
